# Antifraud Medicine QR

[![Build Status](https://github.com/himadro-showrav/antifraud_medicine_qr/actions/workflows/lint-and-tests.yml/badge.svg)](https://github.com/himadro-showrav/antifraud_medicine_qr/actions)

Antifraud Medicine QR is a FastAPI service that helps detect counterfeit medicine distribution by combining encrypted QR payloads with authenticity verification and scan tracking.

It uses ChaCha20-Poly1305 authenticated encryption, PBKDF2 key derivation, and a local issuance registry to validate whether a QR payload was generated by the issuing organization.

## Overview

The platform provides:

- Secure payload encryption for medicine-related data
- QR code generation with optional logo embedding
- Company-issued authenticity metadata (record id, signature, issued timestamp)
- Verification endpoint with duplicate scan detection
- API-first architecture with Swagger and ReDoc
- Dockerized runtime for deployment

## Version Compatibility

The current QR payload format is not guaranteed to be backward compatible with older payloads.

## Architecture Flow

1. Input plaintext and key
2. Derive encryption key via PBKDF2
3. Encrypt payload with ChaCha20-Poly1305
4. Attach issuance metadata and signature
5. Encode payload as QR image
6. Verify on scan using issuance registry and scan count

## Quick Start

### Run with Docker

```bash
docker run --rm -p 8000:8000 himadro-showrav/antifraud_medicine_qr
```

Open:

- Web UI: http://localhost:8000
- Swagger: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## API Endpoints

- POST /v1/encode: Encrypt and generate QR image (company API key required)
- POST /v1/decode: Verify payload metadata and decrypt content
- POST /v1/verify: Public authenticity verification with scan status
- GET /healthz: Service health check

## Example Requests

### Encode Payload

```bash
curl --location 'http://localhost:8000/v1/encode' \
--header 'Content-Type: application/json' \
--data '{
    "plaintext": "medicine batch 123456",
    "key": "my super secret key",
    "company_api_key": "company-encode-key"
}' | jq -r '.content' | base64 --decode > medicine_qrcode.png
```

### Decode Payload

```bash
curl --location 'http://localhost:8000/v1/decode' \
--header 'Content-Type: application/json' \
--data '{
    "encrypted_data": {
        "salt": "LC1bxUNUpMnt/mae1KDNiA==",
        "iterations": 1200000,
        "associated_data": "0WdPVTKSb/a6KjB3NgjFww==",
        "nonce": "FgmmR8D1Su13HgUO",
        "ciphertext": "4FIQ8LAlztzaToyElulDcPAReKGnOd2TFYiH1P9ZatIOuHN+"
    },
    "key": "my super secret key"
}' | jq
```

### Verify Authenticity

```bash
curl --location 'http://localhost:8000/v1/verify' \
--header 'Content-Type: application/json' \
--data '{
    "encrypted_data": {
        "salt": "LC1bxUNUpMnt/mae1KDNiA==",
        "iterations": 1200000,
        "associated_data": "0WdPVTKSb/a6KjB3NgjFww==",
        "nonce": "FgmmR8D1Su13HgUO",
        "ciphertext": "4FIQ8LAlztzaToyElulDcPAReKGnOd2TFYiH1P9ZatIOuHN+",
        "record_id": "issued-record-id",
        "issued_at": "2026-04-05T00:00:00+00:00",
        "signature": "payload-signature"
    }
}' | jq
```

## Configuration

Environment variables use prefix antifraud_medicine_qr_:

- antifraud_medicine_qr_pbkdf2_iterations
- antifraud_medicine_qr_company_api_key
- antifraud_medicine_qr_signing_key

Example:

```bash
docker run --rm -p 8000:8000 \
    -e antifraud_medicine_qr_pbkdf2_iterations=1000000 \
    -e antifraud_medicine_qr_company_api_key=replace-me \
    -e antifraud_medicine_qr_signing_key=replace-me \
    himadro-showrav/antifraud_medicine_qr
```

## Security Notes

- Replace default company and signing keys in production.
- Store keys in a secure secret manager.
- Treat local file-based ledgers as non-distributed storage.
- Use TLS and authenticated upstream gateways in production environments.
